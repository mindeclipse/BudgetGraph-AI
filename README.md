## 📌 Опис застосунку
BudgetGraph — це розумний веб-застосунок, який допомагає відстежувати щоденні витрати, категоризувати їх та контролювати бюджет. Завдяки інтеграції LangGraph-агента, користувачу не потрібно вручну заповнювати форми — достатньо просто написати в чат природною мовою (наприклад, "Витратив 250 грн на обід"), і система автоматично розпізнає суму, категорію та оновить фінансові дашборди.

## 📂 Структура проєкту
* `app.py` — головний файл Streamlit-застосунку (UI, рендеринг компонентів, логіка чату та стрімінгу).
* `agent.py` — модуль з логікою LangGraph-агента, інструментами (tools) та налаштуваннями моделі Gemini 2.5 Flash.
* `.streamlit/secrets.toml` — файл для безпечного зберігання API-ключів (не відстежується в git).
* `requirements.txt` — перелік залежностей для деплою.

## 🛠️ Реалізовані функції та UI-компоненти
1. **Збереження контексту:** Історія повідомлень та стан фінансів зберігаються між оновленнями сторінки завдяки `st.session_state` та `MemorySaver` від LangGraph.
2. **Стрімінг відповідей:** Реалізовано плавний посимвольний вивід відповідей LLM для покращення UX.
3. **Два режими роботи:** Користувач може перемикатися між звичайним чатом та AI-агентом через бічну панель.
4. **Спеціалізовані компоненти Streamlit:**
   * `st.sidebar`: налаштування моделі, лімітів та управління сесією.
   * `st.expander`: компактне згортання аналітичного дашборду.
   * `st.metric` та `st.progress`: візуалізація загальних витрат та контролю ліміту.
   * `st.bar_chart`: автоматична побудова графіка витрат за категоріями.
   * `st.dataframe`: таблиця з історією транзакцій.
5. **Додатково:** кастомна CSS-стилізація та можливість експорту історії чату в JSON (`st.download_button`).

## 🚀 Інструкція з локального запуску

1. Клонуйте репозиторій:
   ```bash
   git clone https://github.com/mindeclipse/BudgetGraph-AI.git
   cd budget_graph_ai
   ```

2. Створіть та активуйте віртуальне середовище:
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Для macOS/Linux
   # або venv\Scripts\activate для Windows
   ```

3. Встановіть залежності:
   ```bash
   pip install -r requirements.txt
   ```

4. Створіть файл .streamlit/secrets.toml та додайте свій API ключ:
   ```bash
   GOOGLE_API_KEY = "ваш_ключ_тут"
   ```

5. Запустіть застосунок:
   ```bash
   streamlit run app.py
   ```

## 💬 Приклади діалогів (Сценарії використання)

Нижче наведено 6 сценаріїв, які демонструють різні аспекти роботи застосунку BudgetGraph.

### 1. Діалог у звичайному режимі чату (без агента)
**Умова:** У бічній панелі обрано режим "Звичайний чат (Gemini)".
***Користувач:** "Привіт! Які основні правила фінансової грамотності ти знаєш? Напиши 3 головні."
***BudgetGraph:** "Привіт! Чудове питання, адже фінансова грамотність – це ключ до спокою та впевненості у майбутньому. Ось 3 головні правила, які я вважаю фундаментом: 1. Контролюй свої доходи та витрати. Це основа основ. Ти не можеш ефективно керувати тим, чого не бачиш. Складай бюджет, відстежуй, куди йдуть твої гроші. Це допоможе тобі бачити повну картину, виявляти "дірки" у бюджеті та приймати обґрунтовані рішення. 2. Заощаджуй та інвестуй регулярно. "Плати собі першому" – це золоте правило. Відкладай частину доходу щомісяця, навіть невелику суму. Створи "фінансову подушку безпеки" (на 3-6 місяців витрат) та подумай про довгострокові інвестиції для досягнення своїх цілей (пенсія, великі покупки, освіта). 3. Розумно керуй боргами. Уникай непотрібних кредитів, особливо з високими відсотками (кредитні картки, мікропозики). Якщо маєш борги, розроби чіткий план їх погашення, починаючи з найдорожчих. Борги можуть бути інструментом (наприклад, іпотека), але тільки якщо ти розумієш, як ними користуватися і можеш їх обслуговувати. Ці три правила – фундамент для побудови стабільного фінансового майбутнього!"

### 2. Діалог з використанням інструментів агента (Додавання даних)
**Умова:** У бічній панелі обрано режим "AI Асистент (LangGraph)".
***Користувач:** "Сьогодні я витратив 150 грн на таксі до офісу та 450 грн на продукти в супермаркеті."
****BudgetGraph:** "Система: Витрату 150.0 грн додано у категорію 'транспорт' (таксі до офісу).
Система: Витрату 450.0 грн додано у категорію 'продукти' (супермаркет)."
*(Відповідь генерується автоматично після виклику інструменту `add_expense_tool`).*

### 3. Діалог з використанням інструментів агента (Аналітика)
* **Користувач:** "Яка в мене зараз загальна сума витрат і скільки я витратив на продукти?"
* **BudgetGraph:** "Загальна сума ваших витрат становить 600.00 грн. З них на продукти ви витратили 450.00 грн."
*(Агент успішно використовує інструменти `total_spent` та `summary_by_category` для формування відповіді).*

### 4. Демонстрація специфічних функцій варіанта (Моніторинг ліміту)
**Умова:** Місячний ліміт встановлено на рівні 5000 грн. Поточні витрати — 600 грн.
* **Користувач:** "Я купив новий монітор за 4600 грн. Запиши це."
* **BudgetGraph:** "Витрату 4600.0 грн додано у категорію 'техніка' (новий монітор).
**Увага:** Ваші загальні витрати (5200.00 грн) перевищили місячний ліміт (5000.00 грн)!"
*(Спрацював вузол `budget_monitor_node`, який перервав стандартну генерацію і додав попередження).*

### 5. Демонстрація UI-компонентів (Вкладка "Детальна аналітика")
**Дія користувача:** Після додавання витрати на монітор (сценарій 4), користувач переходить на вкладку "Детальна аналітика".
**Результат в UI:** * Компонент `st.metric` показує "Витрачено загалом: 5200.00 ₴".
* Компонент `st.progress` заповнений на 100%.
* З'являється червоне текстове попередження `⚠️ Перевищення ліміту на 200.00 ₴!`.
* На графіку `st.bar_chart` з'являється новий великий стовпець "техніка".

### 6. Управління станом через бічну панель (Sidebar)
**Дія користувача:** Користувач змінює параметр "Місячний ліміт" у бічній панелі з 5000 на 6000 грн.
**Результат в UI:** Застосунок миттєво перераховує стан (state) графа. Червоне попередження про перевищення ліміту зникає, а `st.progress` оновлюється і показує, що використано 86% ліміту (5200 з 6000).
